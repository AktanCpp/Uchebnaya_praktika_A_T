**Автор:** Тилегенов Актан Тилегенович

**Дата обновления:** 09.07.2026

---

## Описание проекта

REST API-приложение на базе **FastAPI** (Python) для хранения и поиска информации о станциях экологического мониторинга окружающей среды. База данных — **SQLite**. Фронтенд — полноценный SPA (Single Page Application) на Vanilla JS с Glassmorphism-дизайном, раздаваемый тем же сервером. Поиск и фильтрация работают мгновенно на стороне клиента без перезагрузки страницы.

## Реализованный функционал

### Шаг 1 — Бэкенд ✅

- Подключение к SQLite с автоматическим созданием таблицы при первом запуске
- Автоматическое заполнение БД тестовыми данными (4 станции КР)
- REST API: получение всех станций (`GET /api/stations`)
- REST API: получение станции по ID (`GET /api/stations/{id}`)
- REST API: фильтрация по названию/городу (`?search=Бишкек`)
- REST API: фильтрация по контролируемому параметру (`?parameter=CO2`)
- Защита от SQL-инъекций (параметризованные запросы)
- Настройка CORS
- Автодокументация API (Swagger UI на `/docs`)

### Шаг 2 — Фронтенд: верстка ✅

- Адаптивный SPA-интерфейс в научно-минималистичном стиле
- Sticky header с индикатором статуса подключения к API
- Hero-секция с градиентным фоном и стеклянной панелью поиска (Glassmorphism)
- Поле поиска по названию, городу и адресу
- Фильтрация по параметрам среды через интерактивные чипы (CO₂, PM2.5, SO₂ и др.)
- Сетка карточек станций с Glassmorphism-эффектом и hover-анимациями
- Bootstrap-модальное окно для детального просмотра станции
- Состояния интерфейса: загрузка, пустой результат, ошибка подключения
- Три шрифта строго по назначению: `Montserrat` (заголовки), `Inter` (текст), `JetBrains Mono` (коды и координаты)
- Адаптивность: корректное отображение на мобильных устройствах (от 320px)

### Шаг 3 — Фронтенд: логика ✅

- Архитектура IIFE: весь код изолирован от глобального пространства имён `window`
- Единый объект состояния `state` (список станций, поисковый запрос, активный фильтр)
- Асинхронная загрузка данных через `fetch()` + `async/await` при старте страницы
- Динамический рендеринг карточек станций из JSON-ответа API
- Живой поиск по событию `input` — фильтрация без запросов к серверу
- Переключение фильтров по параметрам через делегирование событий (`Event Delegation`)
- Комбинированная фильтрация: поиск И параметр применяются одновременно
- Динамический счётчик результатов (`Найдено объектов: N`)
- Заполнение модального окна Bootstrap данными выбранной станции
- Обработка всех состояний UI: загрузка → данные / пусто / ошибка
- Кнопки «Сбросить фильтры» и «Повторить попытку» полностью функциональны

## Установка и запуск

```bash
# 1. Создать виртуальное окружение
python -m venv venv

# Windows:
venv\Scripts\activate
# Linux / macOS:
source venv/bin/activate

# 2. Установить зависимости
pip install -r requirements.txt

# 3. Запустить сервер
uvicorn main:app --reload --port 8000
```

После запуска откройте в браузере:

| Адрес                              | Что открывается             |
|------------------------------------|-----------------------------|
| http://localhost:8000/             | Главная страница (фронтенд) |
| http://localhost:8000/api/stations | JSON со всеми станциями     |
| http://localhost:8000/docs         | Интерактивная документация  |

## Примеры API-запросов

```
GET /api/stations                          — все станции
GET /api/stations?search=Бишкек            — поиск по городу
GET /api/stations?parameter=CO2            — фильтр по параметру
GET /api/stations?search=Ош&parameter=NOx  — комбинированный фильтр
GET /api/stations/1                        — станция по ID
```
